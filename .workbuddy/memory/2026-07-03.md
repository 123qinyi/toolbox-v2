## 白屏 Bug 修复 + 复盘
- 问题：点击"质量分析工具"后网页白屏
- 直接原因：数据对比页用了 `AlertCircle` 图标但未在 import 中声明，运行时 ReferenceError，组件树崩溃
- 修复：在 lucide-react import 列表中添加 `AlertCircle`，重新构建部署

### 深层根因（复盘发现）
- **tsc 类型检查实际是空跑**：项目根 `tsconfig.json` 配置了 `"files": []` + project references，但 `package.json` 的 build 脚本是 `tsc --noEmit && vite build`，用的是根 tsconfig
- `tsc --noEmit`（不带 `-b`）遇到 `"files": []` + references 时不检查任何文件，直接 exit 0
- 正确做法：`tsc -b --noEmit` 或 `tsc --noEmit -p tsconfig.app.json`
- **这意味着项目从一开始类型检查就是失效的**，所有 `tsc --noEmit` 的"通过"都是假象
- 用 `tsc --noEmit -p tsconfig.app.json` 验证后发现了 3 个既有错误（包括 ServiceQualityTool 中的类型不匹配）
- 已重新构建（production 模式）并部署

### 教训
1. 新增 JSX 组件/图标引用后，必须手动确认 import 列表
2. 不能盲信 `tsc --noEmit` 通过 = 代码没问题，要确认 tsc 实际在检查文件
3. 应考虑修复 build 脚本为 `tsc -b --noEmit && vite build` 或 `tsc --noEmit -p tsconfig.app.json && vite build`
